# Gaussian Process (GP) 回帰

> 🌐 [English](04-gp.md) | **日本語**

> 関数そのものに事前分布を置く **ノンパラメトリック・ベイズ** 回帰。
> 不確実性 (95% CI) も予測ごとに返却。
> `Hanalyze.Model.GP` モジュール (多入力 + ARD + 多出力対応)。
>
> 関連: [04-kernel.ja.md](04-kernel.ja.md) (固定カーネル) / [04-rff.ja.md](04-rff.ja.md) (大規模近似) /
> 理論: [docs/principles/gp.ja.md](../principles/gp.ja.md)

## 0. 統合 API — `gp` / `gpMulti` (推奨の入口)

GP・カーネルリッジ回帰 (KRR)・Random Fourier Features (RFF) の 3 つは、
**2 軸で尽くされる 1 つの族**である:

|  | 分布あり (`Gp`) | 分布なし (`Krr` 点) |
|---|---|---|
| **厳密** (Gram) | GP = 事後平均 + 事後分散 | KRR = 点予測 |
| **近似** (RFF) | RFF-GP = 近似平均 + 近似分散 | RFF-KRR = 点予測 |

本体は **GP** であり、KRR の予測は GP 事後平均と一致 (λ = σ_n²・平均のみ)、
RFF は両者の低ランク近似にすぎない。よって `Hanalyze.Plot` は
4 象限を 1 つの spec 動詞 `gp` (単変量・`lm` と対称) /
`gpMulti` (多変量・`lmMulti` と対称) に統合する。象限・カーネル・ハイパラ方略を
`GPConfig` 1 つで選ぶ:

```haskell
import Hanalyze.Plot

-- 象限 (seed/D は RFF 近似コンストラクタにだけ載る)
data GPMethod
  = Gp                       -- 厳密 GP   (分布あり → 帯)
  | Krr                      -- 厳密 KRR  (点)
  | GpRff    !Int !Word32    -- RFF 近似 GP  (D 特徴次元, seed)
  | KrrRff !Int !Word32    -- RFF 近似 KRR (D 特徴次元, seed)

-- ハイパラの「決め方 + (固定時のみ) 値」を一箇所に集約
data HyperStrategy
  = FixedHyper GPParams   -- 固定: この値を使う
  | AutoMarginalLik       -- 周辺尤度で自動 (L-BFGS・optimizeGP)
  | AutoCV                -- LOOCV (PRESS) で自動 (Gram-LOOCV)

data GPConfig = GPConfig
  { gpcKernel :: Kernel          -- RBF / Matern52 / Periodic
  , gpcMethod :: GPMethod        -- 象限
  , gpcHyper  :: HyperStrategy   -- ハイパラ方略
  }

defaultGP :: GPConfig            -- = GPConfig RBF Gp AutoMarginalLik

gp      :: GPConfig -> Text   -> Text -> GPSpec        -- 単変量
gpMulti :: GPConfig -> [Text] -> Text -> GPMultiSpec   -- 多変量 (第1予測子を描画軸)
```

データ源 (`ColumnSource`) から `df |-> gp …` で当てはめ、結果は `toPlot` で
そのまま描ける (散布図に重ねるのが定番):

```haskell
-- 厳密 GP (既定): 平滑曲線 + credible 帯
let m = df |-> gp defaultGP "x" "y"
saveSVGBound "gp.svg" (df |>> (layer (scatter "x" "y") <> toPlot m))
```

![厳密 GP (Gp): 事後平均曲線 + credible 帯](../images/gp-exact-ci.svg)

`Krr` 象限は KRR の**点予測**ゆえ帯を出さない (KRR ≡ GP 事後平均・帯を捨てるだけ):

```haskell
let m = df |-> gp (GPConfig RBF Krr AutoMarginalLik) "x" "y"
```

![厳密 KRR (Krr): 点予測のみ・帯なし](../images/gp-ridge-point.svg)

`GpRff D seed` は RFF による低ランク近似。同じ seed なら結果は完全再現 (純粋)、
厳密 GP とほぼ一致しつつ大規模 n で速い:

```haskell
let m = df |-> gp (GPConfig RBF (GpRff 500 42) AutoMarginalLik) "x" "y"
```

![RFF 近似 GP (GpRff): 厳密 GP の低ランク近似 + 帯](../images/gp-rff-ci.svg)

**ハイパラ最適化の露出**: `gpcHyper` で決め方を選ぶ。`AutoMarginalLik` は
周辺尤度最大化 (`optimizeGP`)、`AutoCV` は LOOCV (PRESS) 閉形式。`FixedHyper` で
手動指定も可。

```haskell
df |-> gp (GPConfig RBF Gp AutoCV)               "x" "y"   -- LOOCV でハイパラ選択
df |-> gp (GPConfig Matern52 Gp AutoMarginalLik) "x" "y"   -- カーネルを Matern52 に
df |-> gpMulti defaultGP ["x1", "x2"] "y"                  -- 多変量 (偏依存曲線)
```

**注**: RFF は定常 (シフト不変) カーネルのみ可 (Bochner 定理)。`Periodic` に
RFF 近似象限 (`GpRff`/`KrrRff`) を指定すると、厳密象限へ自動フォールバックする
(`GpRff → Gp` / `KrrRff → Krr`・`log` 通知)。

**⚠ カーネルと `GPParams` フィールドの対応**: カーネルは自分が使うフィールドだけ読み、
無関係なフィールドは**黙って無視**する (型としての不一致はエラーにならない)。

| カーネル | 使う | 無視する |
|---|---|---|
| `RBF` / `Matern52` | `gpLengthScale`, `gpSignalVar` | `gpPeriod`, `gpLengthScales` |
| `Periodic` | + `gpPeriod` | `gpLengthScales` (ARD 非対応) |

- ARD (`gpLengthScales = Just v`) は **多入力経路のみ**適用 (1D `fitGP` / 単変量 `gp` では無視)。
  ベクトル長が入力次元と不一致なら**等方 `gpLengthScale` に安全フォールバック**する。
- **特に注意**: `AutoMarginalLik` / `AutoCV` が最適化するのは `gpLengthScale` /
  `gpSignalVar` / `gpNoiseVar` の 3 つだけ。**`Periodic` の `gpPeriod` は学習されず**、
  初期値 (= データ全幅 `max 0.01 (xMax−xMin)`) のまま固定される。真の周期を使うには
  `FixedHyper (defaultGPParams { gpPeriod = 実周期 })` で明示指定すること。
- `gpPeriod = 0` (Periodic) や `gpLengthScale = 0` はゼロ除算 (NaN/Inf) になる。値は未検証。

以降の節は、この統合の背後にある **低レベル API** (`fitGP` / `optimizeGP` /
`fitGPMV` ほか) の詳細リファレンス。`gp` / `gpMulti` で足りる用途では本節だけ読めばよい。

## 1. モデル

\[ f \sim \text{GP}(0,\ k(x, x'))\quad y_i = f(x_i) + \varepsilon_i,\ \varepsilon_i \sim N(0, \sigma_n^2) \]

事後予測:
\[ \mu(x_*) = k_*^\top (K_y)^{-1} y,\quad
   \sigma^2(x_*) = k(x_*, x_*) - k_*^\top (K_y)^{-1} k_* \]

## 2. API

```haskell
import Hanalyze.Model.GP

-- カーネル
data Kernel = RBF | Matern52 | Periodic

-- ハイパラ
data GPParams = GPParams
  { gpLengthScale  :: Double
  , gpSignalVar    :: Double
  , gpNoiseVar     :: Double
  , gpPeriod       :: Double
  , gpLengthScales :: Maybe (LA.Vector Double)  -- ARD
  }

defaultGPParams    :: GPParams
initParamsFromData :: [Double] -> [Double] -> GPParams

data GPModel  = GPModel  { gpKernel :: Kernel, gpParams :: GPParams }
data GPResult = GPResult { gpTestX, gpMean, gpVar, gpLower, gpUpper :: [Double] }

-- 1D API
fitGP    :: GPModel -> [Double] -> [Double] -> [Double] -> GPResult
fitGPMulti :: ...                                      -- multi-output
optimizeGP :: Kernel -> [Double] -> [Double] -> GPParams -> GPParams

-- 多入力 API (X is n × p)
buildKernelMatrixMV :: Kernel -> GPParams -> Matrix Double -> Matrix Double -> Matrix Double
fitGPMV       :: GPModel -> Matrix Double -> Matrix Double -> Matrix Double -> (Matrix Double, Vector Double)
optimizeGPMV  :: Kernel -> Matrix Double -> Vector Double -> GPParams -> GPParams
```

## 3. ミニマル例 (1D)

```haskell
import qualified Numeric.LinearAlgebra as LA

let xs = [0, 0.1, 0.2, ..., 1.0]
    ys = [...]
    p0 = initParamsFromData xs ys
    pOpt = optimizeGP RBF xs ys p0           -- 周辺尤度最大化
    res = fitGP (GPModel RBF pOpt) xs ys [0, 0.05, ..., 1.0]
-- res.gpMean / gpLower / gpUpper を可視化
```

`fitGP` の結果を描くと、事後平均曲線 (`gpMean`) と予測ごとの 95% 信用区間帯
(`gpLower`〜`gpUpper`) が得られる。観測点の近くで帯が狭まり、データの無い領域で
広がる様子に注目。

![ガウス過程の事後平均曲線と信用区間帯](../images/gp-mean-ci.svg)

## 4. カーネルの選び方

| カーネル | 滑らかさ | 用途 |
|---|---|---|
| `RBF` (Squared exponential) | C∞ | デフォルト、最も滑らか |
| `Matern52` | C⁴ | 物理系、若干粗い関数 |
| `Periodic` | 周期的 | 季節性データ (要 `gpPeriod`) |

## 5. ARD (Automatic Relevance Determination)

`gpLengthScales = Just v` で per-dim length scale を学習。**入力次元の重要度を自動推定** する効果。

```haskell
let p0 = (initParamsFromData ...) { gpLengthScales = Just (LA.konst 1 p) }
    pOpt = optimizeGPMV kern xMat yVec p0
-- pOpt.gpLengthScales が次元ごとの length scale
```

## 6. 多入力 (`fitGPMV`)

```haskell
let xMat = LA.fromLists [[x1_1, x1_2, x1_3], ..., [xn_1, xn_2, xn_3]] :: LA.Matrix Double
    yVec = LA.fromList [y1, ..., yn]
    p0 = initParamsFromData (concat (LA.toLists xMat)) (LA.toList yVec)
    pOpt = optimizeGPMV RBF xMat yVec p0
    (means, vars) = fitGPMV (GPModel RBF pOpt) xMat (LA.asColumn yVec) xTest
```

距離行列は BLAS GEMM で組まれ、reshape も `O(n²p)` の Haskell ループ無し。
詳細: [04-kernel.ja.md](04-kernel.ja.md#6-多次元入力-grammatrixmv--kernelridgemv--nwregressionmv) のセクションも参照。

## 8. 頑健 GP (外れ値に強い)

