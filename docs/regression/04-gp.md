# Gaussian Process (GP) Regression

> 🌐 **English** | [日本語](04-gp.ja.md)

> Non-parametric Bayesian regression that places a prior distribution **over functions themselves**.
> Returns uncertainty (95% CI) for each prediction.
> `Hanalyze.Model.GP` module (multi-input + ARD + multi-output support).
>
> Related: [04-kernel.md](04-kernel.md) (fixed kernel) / [04-rff.md](04-rff.md) (large-scale approximation) /
> Theory: [docs/principles/gp.md](../principles/gp.md)

## 0. Unified API — `gp` / `gpMulti` (recommended entry point)

GP, Kernel Ridge Regression (KRR), and Random Fourier Features (RFF) are three methods
**that form a single family along two axes**:

|  | With distribution (`Gp`) | Without distribution (`Krr` point) |
|---|---|---|
| **Exact** (Gram) | GP = posterior mean + posterior variance | KRR = point prediction |
| **Approximate** (RFF) | RFF-GP = approximate mean + approximate variance | RFF-KRR = point prediction |

The core is **GP**, and KRR prediction matches the GP posterior mean (λ = σ_n² · mean only),
RFF is merely a low-rank approximation of both. Therefore `Hanalyze.Plot` unifies
the 4 quadrants into a single spec verb `gp` (univariate · symmetric to `lm`) /
`gpMulti` (multivariate · symmetric to `lmMulti`). Choose the quadrant, kernel, and hyperparameter strategy
through a single `GPConfig`:

```haskell
import Hanalyze.Plot

-- Quadrant (seed/D appear only in RFF approximation constructors)
data GPMethod
  = Gp                       -- Exact GP   (with distribution → band)
  | Krr                      -- Exact KRR  (point)
  | GpRff    !Int !Word32    -- RFF approximate GP  (D feature dimension, seed)
  | KrrRff !Int !Word32    -- RFF approximate KRR (D feature dimension, seed)

-- Consolidate hyperparameter "decision method + (fixed value only)" in one place
data HyperStrategy
  = FixedHyper GPParams   -- Fixed: use this value
  | AutoMarginalLik       -- Auto by marginal likelihood (L-BFGS · optimizeGP)
  | AutoCV                -- Auto by LOOCV (PRESS) (Gram-LOOCV)

data GPConfig = GPConfig
  { gpcKernel :: Kernel          -- RBF / Matern52 / Periodic
  , gpcMethod :: GPMethod        -- Quadrant
  , gpcHyper  :: HyperStrategy   -- Hyperparameter strategy
  }

defaultGP :: GPConfig            -- = GPConfig RBF Gp AutoMarginalLik

gp      :: GPConfig -> Text   -> Text -> GPSpec        -- Univariate
gpMulti :: GPConfig -> [Text] -> Text -> GPMultiSpec   -- Multivariate (first predictor on display axis)
```

Fit from data source (`ColumnSource`) via `df |-> gp …`, and the result can be plotted directly with `toPlot`
(overlaying on scatter plot is the standard):

```haskell
-- Exact GP (default): smooth curve + credible band
let m = df |-> gp defaultGP "x" "y"
saveSVGBound "gp.svg" (df |>> (layer (scatter "x" "y") <> toPlot m))
```

![Exact GP (Gp): posterior mean curve + credible band](../images/gp-exact-ci.svg)

The `Krr` quadrant shows KRR's **point prediction** only, without a band (KRR ≡ GP posterior mean · band discarded):

```haskell
let m = df |-> gp (GPConfig RBF Krr AutoMarginalLik) "x" "y"
```

![Exact KRR (Krr): point prediction only · no band](../images/gp-ridge-point.svg)

`GpRff D seed` is low-rank approximation by RFF. With the same seed, results are perfectly reproducible (pure),
nearly match exact GP, and are faster for large n:

```haskell
let m = df |-> gp (GPConfig RBF (GpRff 500 42) AutoMarginalLik) "x" "y"
```

![RFF Approximate GP (GpRff): low-rank approximation of exact GP + band](../images/gp-rff-ci.svg)

**Hyperparameter optimization exposure**: Use `gpcHyper` to select the decision method. `AutoMarginalLik` is
marginal likelihood maximization (`optimizeGP`), `AutoCV` is LOOCV (PRESS) closed form. Manual specification is also possible with `FixedHyper`.

```haskell
df |-> gp (GPConfig RBF Gp AutoCV)               "x" "y"   -- Select hyperparams by LOOCV
df |-> gp (GPConfig Matern52 Gp AutoMarginalLik) "x" "y"   -- Change kernel to Matern52
df |-> gpMulti defaultGP ["x1", "x2"] "y"                  -- Multivariate (partial dependence curve)
```

**Note**: RFF applies only to stationary (shift-invariant) kernels (Bochner's theorem). If you specify
an RFF approximation quadrant (`GpRff`/`KrrRff`) with `Periodic`, it automatically falls back to the exact quadrant
(`GpRff → Gp` / `KrrRff → Krr` · log notification).

**⚠ Correspondence between kernel and `GPParams` fields**: The kernel reads only the fields it uses,
and **silently ignores unrelated fields** (type mismatch does not cause an error).

| Kernel | Uses | Ignores |
|---|---|---|
| `RBF` / `Matern52` | `gpLengthScale`, `gpSignalVar` | `gpPeriod`, `gpLengthScales` |
| `Periodic` | + `gpPeriod` | `gpLengthScales` (ARD not supported) |

- ARD (`gpLengthScales = Just v`) applies **only to multi-input path** (1D `fitGP` / univariate `gp` ignores it).
  If vector length does not match input dimension, it **safely falls back to isotropic `gpLengthScale`**.
- **Especially note**: `AutoMarginalLik` / `AutoCV` optimize only the three: `gpLengthScale` /
  `gpSignalVar` / `gpNoiseVar`. **`gpPeriod` in `Periodic` is not learned** and remains fixed at
  the initial value (= data span `max 0.01 (xMax−xMin)`). For the true period, use
  `FixedHyper (defaultGPParams { gpPeriod = actual_period })` to specify explicitly.
- `gpPeriod = 0` (Periodic) or `gpLengthScale = 0` causes division by zero (NaN/Inf). Values are unvalidated.

The sections following this are detailed reference of the **low-level API** (`fitGP` / `optimizeGP` /
`fitGPMV` etc.) underlying the unified interface. For use cases satisfied by `gp` / `gpMulti`, reading this section alone suffices.

## 1. Model

\[ f \sim \text{GP}(0,\ k(x, x'))\quad y_i = f(x_i) + \varepsilon_i,\ \varepsilon_i \sim N(0, \sigma_n^2) \]

Posterior predictive:
\[ \mu(x_*) = k_*^\top (K_y)^{-1} y,\quad
   \sigma^2(x_*) = k(x_*, x_*) - k_*^\top (K_y)^{-1} k_* \]

## 2. API

```haskell
import Hanalyze.Model.GP

-- Kernel
data Kernel = RBF | Matern52 | Periodic

-- Hyperparameters
data GPParams = GPParams
  { gpLengthScale  :: Double
  , gpSignalVar    :: Double
  , gpNoiseVar     :: Double
  , gpPeriod       :: Double
  , gpLengthScales :: Maybe (LA.Vector Double)  -- ARD
  }

defaultGPParams    :: GPParams
initParamsFromData :: [Double] -> [Double] -> GPParams

data GPModel  = GPModel  { gpKernel :: Kernel, gpParams :: GPParams }
data GPResult = GPResult { gpTestX, gpMean, gpVar, gpLower, gpUpper :: [Double] }

-- 1D API
fitGP    :: GPModel -> [Double] -> [Double] -> [Double] -> GPResult
fitGPMulti :: ...                                      -- multi-output
optimizeGP :: Kernel -> [Double] -> [Double] -> GPParams -> GPParams

-- Multi-input API (X is n × p)
buildKernelMatrixMV :: Kernel -> GPParams -> Matrix Double -> Matrix Double -> Matrix Double
fitGPMV       :: GPModel -> Matrix Double -> Matrix Double -> Matrix Double -> (Matrix Double, Vector Double)
optimizeGPMV  :: Kernel -> Matrix Double -> Vector Double -> GPParams -> GPParams
```

## 3. Minimal Example (1D)

```haskell
import qualified Numeric.LinearAlgebra as LA

let xs = [0, 0.1, 0.2, ..., 1.0]
    ys = [...]
    p0 = initParamsFromData xs ys
    pOpt = optimizeGP RBF xs ys p0           -- Maximize marginal likelihood
    res = fitGP (GPModel RBF pOpt) xs ys [0, 0.05, ..., 1.0]
-- Visualize res.gpMean / gpLower / gpUpper
```

Drawing the result of `fitGP` gives the posterior mean curve (`gpMean`) and 95% credible interval band
for each prediction (`gpLower`〜`gpUpper`). Note how the band narrows near observed points and widens
in regions without data.

![Gaussian process posterior mean curve and credible interval band](../images/gp-mean-ci.svg)

## 4. Kernel Selection

| Kernel | Smoothness | Use |
|---|---|---|
| `RBF` (Squared exponential) | C∞ | Default, smoothest |
| `Matern52` | C⁴ | Physical systems, slightly rougher functions |
| `Periodic` | Periodic | Seasonal data (requires `gpPeriod`) |

## 5. ARD (Automatic Relevance Determination)

Learn per-dimension length scales with `gpLengthScales = Just v`. **Automatically estimates input dimension importance**.

```haskell
let p0 = (initParamsFromData ...) { gpLengthScales = Just (LA.konst 1 p) }
    pOpt = optimizeGPMV kern xMat yVec p0
-- pOpt.gpLengthScales is the length scale per dimension
```

## 6. Multi-input (`fitGPMV`)

```haskell
let xMat = LA.fromLists [[x1_1, x1_2, x1_3], ..., [xn_1, xn_2, xn_3]] :: LA.Matrix Double
    yVec = LA.fromList [y1, ..., yn]
    p0 = initParamsFromData (concat (LA.toLists xMat)) (LA.toList yVec)
    pOpt = optimizeGPMV RBF xMat yVec p0
    (means, vars) = fitGPMV (GPModel RBF pOpt) xMat (LA.asColumn yVec) xTest
```

Distance matrices are built by BLAS GEMM, and reshape is O(n²p) without Haskell loops.
Details: also see the section in [04-kernel.md](04-kernel.md#6-multi-dimensional-input-grammatrixmv--kernelridgemv--nwregressionmv).

## 7. Alternatives for Large n

| n | Recommended |
|---|---|
| ≤ 1000 | `fitGPMV` (exact GP) |
| 1000-10000 | [`fitRFF`](04-rff.md) (Random Fourier Features approximation) |
| > 10000 | sparse GP (to be added in future) |

## 8. Robust GP (resistant to outliers)

StudentT / Cauchy observation likelihood in `Hanalyze.Model.GPRobust`.

```haskell
import Hanalyze.Model.GPRobust
let fit = fitGPRobust RBF p0 (RStudentT 4.0 0.1) xs ys
```
