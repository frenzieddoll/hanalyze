## 1. LM とは

**線形回帰モデル** は応答 $y$ を説明変数 $\mathbf{x} = (x_1, \ldots, x_p)$
の **線形結合 + 正規ノイズ** で表現:

$$ y_i = \beta_0 + \beta_1 x_{i1} + \cdots + \beta_p x_{ip} + \varepsilon_i, \quad \varepsilon_i \sim \text{Normal}(0, \sigma^2) $$

行列で書くと:

$$ \mathbf{y} = X\boldsymbol\beta + \boldsymbol\varepsilon $$

- $\mathbf{y} \in \mathbb{R}^n$: 応答 (n 観測)
- $X \in \mathbb{R}^{n \times (p+1)}$: 計画行列 (1 列目は切片用 1 のベクトル)
- $\boldsymbol\beta \in \mathbb{R}^{p+1}$: 係数 (切片 $\beta_0$ + 傾き $\beta_1, \ldots, \beta_p$)
- $\boldsymbol\varepsilon \in \mathbb{R}^n$: 残差 (誤差項)

### 「線形」の意味

「**$\boldsymbol\beta$ について線形**」が肝心です:

- ✅ $y = \beta_0 + \beta_1 x + \beta_2 x^2$  (x について非線形だが β について線形)
- ✅ $y = \beta_0 + \beta_1 \log x$
- ✅ $y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \beta_3 (x_1 \cdot x_2)$
- ❌ $y = \beta_0 + \exp(\beta_1 x)$  (β について非線形)

つまり、x を変換した特徴量を作れば、**多項式・指数・対数・相互作用項**
すべて LM で表現可能。

## 2. モデルの仮定

LM の OLS 推定が最良 (BLUE = 最良線形不偏推定量) になるには
以下の **4 仮定 (Gauss-Markov 条件)** が必要です:

| # | 仮定 | 内容 | 違反するとどうなるか |
|---|---|---|---|
| 1 | **線形性** | $E[y \mid \mathbf{x}] = X\boldsymbol\beta$ | bias (推定が真の関数からずれる) |
| 2 | **独立性** | $\text{Cov}(\varepsilon_i, \varepsilon_j) = 0$ for $i \ne j$ | 標準誤差の過小評価 |
| 3 | **等分散性** | $\text{Var}(\varepsilon_i) = \sigma^2$ (一定) | 標準誤差の歪み (= 推論が不正確) |
| 4 | **正規性** | $\varepsilon_i \sim \text{Normal}(0, \sigma^2)$ | 信頼区間・p 値が不正確 (大標本では緩和) |

### 補足
- **正規性は推定の一致性には不要**。あくまで信頼区間・仮説検定の正確性のため。
- **等分散性は重要**。違反すると **GLS / WLS** や **robust standard errors** で対処。
- **独立性違反の例**: 時系列データ (自己相関)、グループ構造 (兄弟・学校・地域)。
  - 時系列なら AR 誤差項、グループなら **GLMM** (Phase 03) で対処。

## 3. 最小二乗法 (OLS) の導出

### 3.1 残差平方和の最小化

OLS は以下を最小化:

$$ \text{RSS}(\boldsymbol\beta) = \|\mathbf{y} - X\boldsymbol\beta\|^2 = (\mathbf{y} - X\boldsymbol\beta)^T (\mathbf{y} - X\boldsymbol\beta) $$

$\boldsymbol\beta$ で偏微分して 0 とおく:

$$ \frac{\partial \text{RSS}}{\partial \boldsymbol\beta} = -2 X^T (\mathbf{y} - X\boldsymbol\beta) = 0 $$

これを解くと **正規方程式** $X^T X \boldsymbol\beta = X^T \mathbf{y}$、つまり:

$$ \boxed{\hat{\boldsymbol\beta} = (X^T X)^{-1} X^T \mathbf{y}} $$

### 3.2 hanalyze の実装

`Hanalyze.Model.LM.fitLM` は内部で **QR 分解** を使う (hmatrix の `<\>`):

```haskell
-- src/Model/LM.hs より抜粋
fitLM :: LA.Matrix Double -> LA.Matrix Double -> FitResult
fitLM x y =
  let beta  = x LA.<\> y                   -- p × q
      yHat  = x LA.<> beta                 -- n × q
      resid = y - yHat
      r2    = computeR2Multi y yHat
  in FitResult beta yHat resid r2
```

**QR 分解** は $X = QR$ (Q は直交、R は上三角) で:
- $(X^T X)^{-1} X^T \mathbf{y} = R^{-1} Q^T \mathbf{y}$
- 数値的に **逆行列より安定** (Cholesky / LU より優位)

### 3.3 Hat 行列と影響度

予測値 $\hat{\mathbf{y}} = X\hat{\boldsymbol\beta} = X(X^T X)^{-1} X^T \mathbf{y} = H\mathbf{y}$

$H = X(X^T X)^{-1} X^T$ は **hat 行列** で、

- 対称: $H = H^T$
- べき等: $H^2 = H$
- $\text{tr}(H) = p + 1$ (パラメタ数)

対角成分 $h_{ii}$ は **leverage** (= 観測 $i$ が自分の予測値に与える影響)。
高 leverage は **外れ値** や **影響点** の検出に使う ($> 2(p+1)/n$ なら警戒)。

## 5. 信頼区間と予測区間

### 5.1 推定量の分散

OLS 推定量の分散:

$$ \text{Var}(\hat{\boldsymbol\beta}) = \sigma^2 (X^T X)^{-1} $$

$\sigma^2$ の不偏推定量:

$$ \hat\sigma^2 = \frac{\text{RSS}}{n - p - 1} = \frac{\|\mathbf{y} - X\hat{\boldsymbol\beta}\|^2}{n - p - 1} $$

各係数の標準誤差 $\text{SE}(\hat\beta_j) = \sqrt{\hat\sigma^2 [(X^T X)^{-1}]_{jj}}$。

### 5.2 平均応答の信頼区間 (CI)

新しい点 $\mathbf{x}_*$ で予測値 $\hat y_* = \mathbf{x}_*^T \hat{\boldsymbol\beta}$ の **平均** に対する区間:

$$ \hat y_* \pm t_{\alpha/2, n-p-1} \cdot \hat\sigma \sqrt{\mathbf{x}_*^T (X^T X)^{-1} \mathbf{x}_*} $$

### 5.3 予測区間 (PI)

**個別の新観測** $y_*$ に対する区間 (PI > CI):

$$ \hat y_* \pm t_{\alpha/2, n-p-1} \cdot \hat\sigma \sqrt{1 + \mathbf{x}_*^T (X^T X)^{-1} \mathbf{x}_*} $$

PI が CI より広いのは、観測誤差 $\sigma^2$ の追加分があるため。

### 5.4 hanalyze での計算

```haskell
import Model.LM (confidenceBand, fitWithCI, fitPolyWithSmooth)
import Model.Core (Band (..))

-- 信頼帯
let (fit, ciBand) = case fitWithCI 0.95 df "x" "y" of
      Just r -> r
      Nothing -> error "fit failed"

-- スプライン風滑らかな fit + CI/PI
let Just (fit, smooth) = fitPolyWithSmooth (CI 0.95) 100 df "x" "y"
-- smooth has sfFit, sfLower, sfUpper for plotting
```

## 6. 診断 (Diagnostics)

### 6.1 R² (決定係数)

$$ R^2 = 1 - \frac{\text{RSS}}{\text{TSS}} = 1 - \frac{\sum (y_i - \hat y_i)^2}{\sum (y_i - \bar y)^2} $$

- $R^2 \in [0, 1]$、1 に近いほど fit が良い
- **欠点**: 説明変数を増やすほど自動的に増える (= 過学習に鈍感)
- **対策**: 自由度調整済み $R_{\text{adj}}^2 = 1 - \frac{\text{RSS}/(n-p-1)}{\text{TSS}/(n-1)}$

### 6.2 残差プロット

`fitted vs residuals` プロットを描き、以下を確認:

| パターン | 示唆 |
|---|---|
| ランダムに散らばる | 良好 (仮定 OK) |
| ファネル形 (fan) | 不等分散 (heteroscedasticity) |
| 曲線傾向 | 線形性違反 (非線形項を追加検討) |
| 周期性 | 自己相関 (時系列・グループ構造) |

### 6.3 QQ プロット

残差を理論的正規分布の分位点と比較。直線に乗れば正規性 OK。
重い裾 / 左右非対称 / 外れ値が見える。

### 6.4 Cook's distance

観測 $i$ を除いた時の係数変化量:

$$ D_i = \frac{r_i^2}{p \hat\sigma^2} \cdot \frac{h_{ii}}{(1-h_{ii})^2} $$

- $D_i > 1$ で **影響点** の警戒 (Cook 1977)
- もしくは $D_i > 4/n$ を目安

### 6.5 hanalyze での診断

```haskell
import Model.Core (residualsV, fittedV, rSquared1)

let r2  = rSquared1 fit
    res = residualsV fit
    yh  = fittedV fit

-- res と yh をペアにして散布図 (Viz.Scatter で)
-- 自己相関は手動: corr(res[1:], res[:-1])
```

CLI なら `--report` で HTML レポート (R²・残差・QQ・予測曲線) を一括生成 (現在は legacy `Hanalyze.Viz.AnalysisReport` 経由。新規実装は `Hanalyze.Viz.ReportBuilder` を使うこと):

```bash
hanalyze data.csv x y LM --ci 0.95 --report
```

### 6.6 推論統計量 (`Hanalyze.Model.LM.Diagnostics`)

`Hanalyze.Model.LM.Diagnostics` は **多出力標準** (`Matrix p × q` 返し) で SE / t / p、
F-statistic、AIC / BIC、leverage、Cook's distance を提供する。1 出力時は
薄い `Vector` ラッパで使える。

```haskell
import qualified Model.LM             as LM
import qualified Model.LM.Diagnostics as LMD

let x    = LM.designMatrix xVec    -- n × p (intercept 列込み)
    fit  = LM.fitLMVec x y

    -- 1 出力 (q = 1) 用ラッパ
    ses     = LMD.lmStdErrors x fit            -- Vector p
    cs      = LMD.lmCoefStats x fit            -- [CoefStats] (長さ p)
    fStat   = head (LMD.lmFStatistic x fit)    -- FStat
    ic      = LMD.lmInformationCriteria fit    -- ICs
    levH    = LMD.hatDiagonal x                -- Vector n
    rTilde  = LMD.standardizedResiduals x fit  -- Vector n
    cooksD  = LMD.cooksDistance x fit          -- Vector n
    sdX     = LMD.predictorStdDevs x           -- Vector p (intercept 列は 0)
    tCrit   = LMD.ciTValue 0.95 (LA.rows x - LA.cols x)

    -- 多出力 (q > 1) 用 (主 API)
    seMat   = LMD.lmStdErrorsMulti x fitMulti  -- Matrix p × q
    csList  = LMD.lmCoefStatsMulti x fitMulti  -- [[CoefStats]] (q × p)
```

`CoefStats` は `csSE / csTValue / csPValue`、`FStat` は `fsValue / fsPValue
/ fsDf1 / fsDf2`、`ICs` は `icLogLik / icAIC / icBIC` を持つ。AIC / BIC は
**R `lm()` の慣例** に合わせ `k = p + 1` (σ も自由パラメタとして数える) を採用
(statsmodels は `k = p` で 2 / `log n` の差が出る)。

## 7. 仮定の検証と対処

### 7.1 等分散性違反 (Heteroscedasticity)

**徴候**: 残差プロットがファネル形 (x 大で残差広がる)。

**対処**:

1. **応答を変換**: $\log y$, $\sqrt{y}$ で分散安定化
2. **重み付き最小二乗 (WLS)**: 各観測に逆分散重み $1/\sigma_i^2$
3. **GLM with appropriate variance function** ([02-glm.ja.md](02-glm.ja.md))
4. **Robust standard errors** (= "sandwich estimator", hanalyze 未実装)

### 7.2 正規性違反

**徴候**: QQ プロットが直線から外れる、特に裾。

**対処**:

1. 大標本なら問題なし (中心極限定理で頑健)
2. **応答変換**: $\log y$, Box-Cox 変換
3. **Robust regression**: M-estimator, Huber 損失 (hanalyze 未実装)
4. **Bayes 化**: `Hanalyze.Model.HBM` で StudentT(ν) 観測 → 重い裾を許容

### 7.3 独立性違反 (相関)

**徴候**: Durbin-Watson < 1.5 or > 2.5 (時系列)、グループ内分散 ≠ グループ間。

**対処**:

1. **時系列**: AR 誤差項 (hanalyze は `ar1Latent` で Bayes 化可能)
2. **グループ構造**: `Hanalyze.Model.GLMM.fitLME` ([03-glmm.ja.md](03-glmm.ja.md))
3. **クラスタロバスト SE**: hanalyze 未実装

### 7.4 線形性違反

**徴候**: 残差プロットが曲線傾向。

**対処**:

1. **多項式項を追加**: `polyDesignMatrix 2 xs` で 2 次まで
2. **Spline**: [04-regularized.ja.md](04-regularized.ja.md)
3. **GAM** (Generalized Additive Models, hanalyze 未実装)
4. **GLM with log link**: 指数的関係には [02-glm.ja.md](02-glm.ja.md)

## 8. カテゴリ変数とダミー化

### 8.1 ダミー変数

**カテゴリ変数 (例: 性別, 国, 製品種類)** は数値化が必要:

| 元データ | 男 | 女 | その他 |
|---|---|---|---|
| 男 | 1 | 0 | 0 |
| 女 | 0 | 1 | 0 |
| その他 | 0 | 0 | 1 |

ただし **基準カテゴリ** を 1 つ落として multi-collinearity を避ける:

| | dummy_F | dummy_O |
|---|---|---|
| 男 (基準) | 0 | 0 |
| 女 | 1 | 0 |
| その他 | 0 | 1 |

→ 切片が「男の平均」、係数が「女 vs 男の差」「その他 vs 男の差」。

### 8.2 hanalyze での実装

現在の `Hanalyze.Model.LM` には自動ダミー化がないため、ユーザーが計画行列を構築:

```haskell
-- 簡易: 3 カテゴリを 2 ダミーに
let isFemale = LA.fromList [if g == "F" then 1 else 0 | g <- groups]
    isOther  = LA.fromList [if g == "O" then 1 else 0 | g <- groups]
    dm = LA.fromColumns [LA.konst 1 n, isFemale, isOther, LA.fromList xs]
```

### 8.3 Treatment / Sum / Helmert コーディング

- **Treatment** (デフォルト): 1 つを基準に
- **Sum**: 各係数 = カテゴリ平均 - 全体平均
- **Helmert**: 連続的に対比 (順序カテゴリで使う)

hanalyze は明示的サポートなし。手動で計画行列を組む。

## 9. 多項式回帰と相互作用項

### 9.1 多項式回帰

```haskell
import Model.LM (polyDesignMatrix)

let dm = polyDesignMatrix 3 xs   -- [1, x, x², x³]
    fit = fitLMVec dm ys
```

**注意**: 高次多項式は **過学習** + **境界外で発散**。3-4 次で打ち止め推奨。
非線形な関係なら **スプライン** ([04-regularized.ja.md](04-regularized.ja.md)) を推奨。

### 9.2 相互作用項

「効果がもう 1 つの変数に依存する」場合:

$$ y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \beta_3 (x_1 \cdot x_2) + \varepsilon $$

例: x1 = 性別ダミー、x2 = 経験年数。$\beta_3$ は「経験 1 年あたりの効果が
性別でどう違うか」。

```haskell
-- 手動で相互作用項を計算
let x12 = LA.fromList [a * b | (a, b) <- zip xs1 xs2]
    dm  = LA.fromColumns [LA.konst 1 n, x1V, x2V, x12]
```

### 9.3 中央化 (Centering)

多項式・相互作用項では **x を平均で中央化** (= $x - \bar x$) すると条件数が改善:

```haskell
let xMean = sum xs / fromIntegral (length xs)
    xCent = [x - xMean | x <- xs]
```

## 10. 多重共線性

### 10.1 何が問題か

**説明変数同士が強く相関** すると:

- $X^T X$ が ill-conditioned (条件数 > 30 で警戒)
- 個別係数の標準誤差が膨らむ
- 推定が不安定 (データを少し変えると大きく変動)
- 解釈不能 (どの変数が効いているか分からない)

### 10.2 検出: VIF (Variance Inflation Factor)

各列について「他の列で回帰した時の $R_j^2$」から:

$$ \text{VIF}_j = \frac{1}{1 - R_j^2} $$

| VIF | 評価 |
|---|---|
| < 5 | OK |
| 5-10 | 注意 |
| > 10 | 深刻 (実用上問題) |

```haskell
import Design.Quality (vifList)

let vifs = vifList xRows   -- 各列の VIF
```

### 10.3 対処

1. **相関の高い変数を 1 つ削除**
2. **PCA** で直交化してから回帰 (PCR)
3. **PLS** (= y との共分散最大方向)
4. **Ridge** 正則化 (= L2 で安定化、[04-regularized.ja.md](04-regularized.ja.md))
5. **中央化** (多項式項でだけ有効)

### 10.4 条件数

$$ \kappa = \lambda_{\max} / \lambda_{\min} \quad (X^T X \text{の固有値}) $$

- $\kappa < 100$: 良好
- $\kappa > 1000$: 深刻

```haskell
import Design.Quality (conditionNumber)
let kappa = conditionNumber xRows
```

## 11. モデル比較

### 11.1 古典的指標

| 指標 | 定義 | 解釈 |
|---|---|---|
| **R²** | 1 - RSS/TSS | 1 に近いほど fit 良い |
| **R²_adj** | df 補正済み | 過学習にやや頑健 |
| **AIC** | 2k - 2 log L | 小さいほど良い |
| **BIC** | k log n - 2 log L | AIC より罰則強 |
| **Mallows Cp** | (RSS + 2 p σ̂²)/n | 小さいほど良い |

`hanalyze` には AIC/BIC 関数は明示的にないが、`logLikelihood` から算出可能。

### 11.2 ベイズ的指標 (推奨)

- **WAIC** (Widely Applicable Information Criterion)
- **PSIS-LOO** (Pareto Smoothed Importance Sampling LOO-CV)

```haskell
import Stat.ModelSelect (waic, loo, lmPosteriorLogLiks, compareModels, ModelInfo (..))

-- LM の posterior をフラット事前で生成
loglik <- lmPosteriorLogLiks x y fit 1000 gen
let waicResult = waic loglik
    looResult  = loo  loglik

-- 複数モデル比較
let cmps = compareModels [ ModelInfo "M1" loglik1
                         , ModelInfo "M2" loglik2 ]
```

詳細は [bayesian/06-model-comparison.ja.md](../bayesian/06-model-comparison.ja.md)。

### 11.3 交差検証

k-fold CV で out-of-sample MSE を評価:

```haskell
-- hanalyze には組込み CV ヘルパなし、手動で:
let folds = splitIntoFolds 5 data
    mses  = [ mseOnFold f folds | f <- [0..4] ]
```

## 12. よくある落とし穴

### 12.1 切片を忘れる

`designMatrix` を使えば自動だが、自作の場合は **必ず 1 列目に 1 の列を入れる**:

```haskell
-- ❌ 切片なし
let dm = LA.fromColumns [xV1, xV2]

-- ✅ 切片あり
let dm = LA.fromColumns [LA.konst 1 n, xV1, xV2]
```

### 12.2 データのスケーリング

xs のオーダーが違いすぎる (例: 年齢 ∈ [0, 100] と所得 ∈ [0, 10⁷]) と:

- 条件数悪化
- 数値計算で精度低下
- 係数の絶対値比較が無意味

→ **z-score 標準化** (`Hanalyze.Model.Regularized.standardize`) や log 変換で揃える。

### 12.3 外挿 (Extrapolation)

訓練範囲外の x で予測すると **大きく外れる** ことがある。
範囲を確認し、警告するか拒否する。

### 12.4 「相関 ≠ 因果」

回帰の係数は **条件付き相関** であって因果効果ではない:

- 交絡因子 (confounder) があると bias
- 内生性 (endogeneity) があると bias
- 因果推論には DoWhy / IV / DID 等の道具が要る (hanalyze 範囲外)

### 12.5 過学習

n が p (特徴量数) に近い場合、訓練 R² は 1 に近づくが汎化性能は悪化。

→ **Lasso / Ridge** 正則化 ([04-regularized.ja.md](04-regularized.ja.md))。

### 12.6 Simpson のパラドックス

集約データで見た傾向が **グループ別では逆転する** 現象。

```bash
cabal run simpson-paradox  # デモ
```

→ **混合効果 (LME/GLMM)** で対処 ([03-glmm.ja.md](03-glmm.ja.md))。
