# Stat.CV — Cross-Validation

> 🌐 **English** | [日本語](04-cv.ja.md)

> Cross-validation framework equivalent to sklearn.model_selection.

## 1. Split Strategies

```haskell
-- Random k-fold (each fold size = n/k)
folds <- CV.kFold 5 100 gen

-- Preserve class balance
folds <- CV.stratifiedKFold 5 labels gen

-- Leave-one-out (n folds)
folds <- CV.leaveOneOut 100

-- Monte-Carlo CV
folds <- CV.shuffleSplit 10 0.2 100 gen  -- 10 reps, 20% test

-- Time series (forward-chaining)
let folds = CV.timeSeriesSplit 50 10 100  -- initial=50, step=10, n=100
```

## 2. General CV Loop

```haskell
import qualified Hanalyze.Stat.CV as CV

scores <- CV.crossValidate folds fitFn scoreFn dataset
  where
    fitFn (trainIdx, testIdx) data = do
      -- fit model on train, predict on test
      pure predictions
    scoreFn data testIdx pred = do
      -- return evaluation metric
      pure 0.95

-- mean ± std
(mean, std) <- CV.crossValidateScores folds fitFn scoreFn dataset
```

## 3. Grid Search (Hyperparameter Exploration)

```haskell
let grid = [(lambda, gamma) | lambda <- [0.01, 0.1, 1.0],
                              gamma  <- [0.5, 1.0, 2.0]]

result <- CV.gridSearchCV folds grid fitWithHP scoreFn dataset

CV.gsBestParams result   -- (0.1, 1.0)
CV.gsBestScore result    -- 0.847
CV.gsAllResults result   -- list of all (hp, mean, std)
```

## 4. Strategy Selection Guide

| Scenario | Recommended |
|---|---|
| Standard regression/classification | `kFold` (k = 5 or 10) |
| Imbalanced classification | `stratifiedKFold` |
| Small data (n < 50) | `leaveOneOut` |
| Time series | `timeSeriesSplit` (forward-chaining) |
| Unstable model | `shuffleSplit` (repeated) |

## 5. Cautions

- **Data leak prevention**: Preprocess (scaling, encoding) only on train fold
- **Nested CV**: Separate CV for hyperparameter selection and performance evaluation
