Supervised learning: trees, boosting, neighbor methods. Fit on matrices and render results with `toPlot` (classifiers' decision boundaries and confusion matrices use generic `decisionBoundaryOf` / `confusionOf`). Theory is documented in [ml/usage-ml-extensions](../ml/usage-ml-extensions.md) and [08-decisiontree](../regression/08-decisiontree.md).

High-level is **`df |-> spec`**. Each spec takes a **feature column list `[Text]`** and a **class / response column `Text`**. Following scikit-learn, we affix **`Cls` for classification and `Reg` for regression** to make both explicit (`randomForestReg` / `randomForestCls`, `gbmReg` / `gbmCls`, `knnReg` / `knnCls` …); **no unaffixed versions** are created. Only techniques with a single role omit the affix (`decisionTree` / `naiveBayes` are classification-only). Pure-fit methods (GBM / decision tree / k-NN / NB) use `df |->` directly; Random Forest and NN, which use RNG, take a seed spec for purification (see below).

| Technique | High-level (`df \|->`) | Result type | Plot |
|---|---|---|---|
| Gradient boosting (regression) | `df \|-> gbmReg cfg featCols yCol` | `GBRegressor` (Plottable) | importance bar |
| Gradient boosting (classification) | `df \|-> gbmCls cfg featCols clsCol` | `GBClassifier` (Plottable) | importance bar |
| Decision tree | `df \|-> decisionTree cfg featCols clsCol` | `DTFit` (Plottable) | tree structure (rpart.plot style) |
| k-NN (classification) | `df \|-> knnCls k featCols clsCol` | `KNNClassifier` (Plottable) | decision boundary / confusion matrix |
| k-NN (regression) | `df \|-> knnReg k featCols yCol` | `KNNRegressor` | — |
| Naive Bayes | `df \|-> naiveBayes featCols clsCol` | `NBModel` (Plottable) | decision boundary |
| Random Forest (regression) | `df \|-> randomForestReg cfg seed featCols yCol` | `RandomForest` (Plottable) | importance 2 panels (impurity/permutation) |
| Random Forest (classification) | `df \|-> randomForestCls cfg seed featCols clsCol` | `RFClassifierFit` (Plottable) | importance 2 panels (permutation/gini), OOB error |
| Neural network (classic MLP, classification) | `df \|-> mlpCls cfg seed featCols clsCol` | `MLPFit` | loss curve / decision boundary |
| Neural network (classic MLP, regression) | `df \|-> mlpReg cfg seed featCols yCol` | `MLPFit` | loss curve |
| SVM (classification, kernel selection, CV tuning) | `df \|-> svmCls cfg featCols clsCol` | `SVMMulti` | decision boundary (nonlinear) / confusion matrix |
| MDS (unsupervised dimension reduction) | `df \|-> mds cfg featCols` | `MDSResult` (Plottable) | embedding scatter |

> - **`featCols :: [Text]`** = feature (predictor) column names. **`clsCol` / `yCol :: Text`** =
>   class column for classification / response column for regression (single).
>   Class columns are rounded to integer internally; response columns remain real.
> - Low-level matrix APIs (`fitGBRegressor` / `fitDTV` / `fitKNNC` / `fitGNB` etc) are noted in each section.

## Random Forest

Regression via `randomForestReg`, classification via `randomForestCls`. **Real-valued response = regression; class column = classification.**
Both fit via `df |->`, take a seed (for per-tree bootstrap and feature sampling randomness), and produce plots in the R `varImpPlot` style: **two importance panels** (descending, real column names, horizontal bars). Like `gbmReg` / `gbmCls`, regression and classification use symmetric `Reg` / `Cls` naming (no unaffixed version):

```haskell
randomForestReg :: RFConfig  -> Word32 -> [Text] -> Text -> RFSpec    -- regression → RandomForest    (Plottable)
randomForestCls :: RFCConfig -> Word32 -> [Text] -> Text -> RFCSpec   -- classification → RFClassifierFit (Plottable)
-- args: config / random seed / feature columns / response column (regression=real yCol; classification=class clsCol)
```

Config types differ between regression and classification (field names also differ):

```haskell
data RFConfig = RFConfig          -- defaultRandomForest (regression)
  { rfTrees      :: Int           -- number of trees              (default 100)
  , rfMaxDepth   :: Int           -- tree depth limit             (default 12)
  , rfMinSamples :: Int           -- min samples to split         (default 3)
  , rfMtry       :: Maybe Int     -- candidate features per split (Nothing for √p; default Nothing)
  , rfBootstrap  :: Bool          -- use bootstrap samples        (default True)
  }

data RFCConfig = RFCConfig        -- defaultRFCConfig (classification)
  { rfcNTrees   :: Int            -- number of trees              (default 100)
  , rfcMaxDepth :: Maybe Int      -- depth limit (Nothing for unlimited; default Just 10)
  , rfcMinSplit :: Int            -- min samples to split         (default 2)
  }
```

Fit both regression and classification, producing importance plots for each:

```haskell
let randomForestRegModel = df |-> randomForestReg defaultRandomForest 42 ["area","age","noise"] "price"    -- RandomForest
    randomForestClsModel = df |-> randomForestCls defaultRFCConfig    42 ["x1","x2","x3"]       "species"  -- RFClassifierFit
saveSVG "rf-importance.svg"  $ toPlot randomForestRegModel
saveSVG "rfc-importance.svg" $ toPlot randomForestClsModel
```

![RandomForest feature importance (regression)](../images/rf-importance.svg)

![RandomForest feature importance (classification)](../images/rfc-importance.svg)

**Regression** 2 panels: **impurity (IncNodePurity)** and **permutation (%IncMSE)**. Impurity tends to overweight high-cardinality features, so permutation is included. Prediction and accessors = `predictRF` / `featureImportance` (impurity) / `rfPermutationImportance`.

**Classification** 2 panels: **permutation (MeanDecreaseAccuracy)** and **gini reduction (MDI)**. Prediction and accessors = `predictRFClassifier` / `rfcOOBError` (OOB error rate) / `rfcClasses`.

## Gradient boosting (GBM)

Regression via `gbmReg`, classification via `gbmCls`. Both fit via `df |->` and produce feature importance bar plots:

```haskell
gbmReg :: GBConfig -> [Text] -> Text -> GBRSpec   -- regression → GBRegressor  (Plottable)
gbmCls :: GBConfig -> [Text] -> Text -> GBCSpec   -- classification → GBClassifier (Plottable)
```

Config `GBConfig` (shared between regression and classification):

```haskell
data GBConfig = GBConfig          -- defaultGBM
  { gbNRounds    :: Int           -- boosting rounds M            (default 100)
  , gbMaxDepth   :: Int           -- weak learner depth (typical 3-5; default 3)
  , gbMinSamples :: Int           -- min leaf samples             (default 2)
  , gbLearnRate  :: Double        -- learning rate η (typical 0.1; default 0.1)
  }
```

```haskell
let gbmRegModel = df |-> gbmReg defaultGBM ["x1","x2"] "y"   -- GBRegressor
saveSVG "gbm-importance.svg" $ toPlot gbmRegModel
```

![GBM feature importance](../images/gbm-importance.svg)

**Low-level**: `fitGBRegressor :: GBConfig -> Matrix Double -> VU.Vector Double -> GBRegressor` /
`fitGBClassifier :: … -> VU.Vector Int -> GBClassifier`. Prediction via `predictGBR` / `predictGBC` /
`predictGBCProbs` (class labels / probabilities).

## Partial dependence (PDP / ICE)

Grid a feature of interest; hold other features at training distribution and predict each row, then average (R `pdp::partial` / sklearn `PartialDependenceDisplay` equivalent; model-agnostic). Applied to trained tree-ensemble models (`RandomForest` / `GBRegressor`).

PDP is a **Plottable intermediate type** (`pdp`), like HBM extractors (`forestOf` etc), rendered with `toPlot` and decorated with `<>`. `featCols` = features used in fit; the last arg = column to examine:

```haskell
-- randomForestRegModel = df |-> randomForestReg … ["area","age","noise"] "price" trained
saveSVGBound "pdp-area.svg" $
  noDf |>> toPlot (pdp randomForestRegModel trainDf ["area","age","noise"] "area")
           <> title "Partial dependence: price ~ area"
```

![Partial dependence plot PDP](../images/pdp-area.svg)

`pdpIce` overlays individual curves (ICE) in light gray, with the average (PDP) on top (sklearn `kind='both'`). Parallel curves indicate weak feature interactions:

```haskell
saveSVGBound "pdp-ice-area.svg" $
  noDf |>> toPlot (pdpIce randomForestRegModel trainDf ["area","age","noise"] "area")
           <> title "ICE + PDP: price ~ area"
```

![PDP + ICE individual curves](../images/pdp-ice-area.svg)

> HBM fit is self-contained with an internal posterior; RF/GBM do not retain training data, so PDP requires training df for marginalization (the training distribution is needed). The old `pdpOf model df featCols target` / `pdpIceOf` returning `VisualSpec` directly also remain for backward compatibility.

**Low-level**: `pdpPlot` / `pdpIcePlot` take matrices and column indices directly; pure kernel `partialDependence`
(`Hanalyze.Model.PartialDependence` → `PDPResult`).

## Decision tree

Classification only:

```haskell
decisionTree :: DTConfig -> [Text] -> Text -> DTSpec   -- classification → DTFit (Plottable)
```

Config `DTConfig` (sklearn-compatible defaults):

```haskell
data DTConfig = DTConfig          -- defaultDecisionTree
  { dtMaxDepth        :: Maybe Int  -- depth limit (Nothing for unlimited; default Nothing)
  , dtMinSamplesSplit :: Int        -- min samples to split               (default 2)
  , dtMinSamplesLeaf  :: Int        -- min leaf samples                  (default 1)
  , dtMinImpurity     :: Double     -- min impurity decrease to allow split (default 0)
  }
```

`df |-> decisionTree` returns `DTFit` (tree + feature names + class level names). Class column can be **either factor (text) or numeric**; if text, level names appear directly in plots as class names (numeric appears as `0`/`1`/…). Rendering via `toPlot` (R `rpart.plot` style) shows, at each node, predicted class, all-class probabilities, sample fraction, colors by class per ColorBrewer darkened by probability, split condition, `yes`/`no` branches, and legend—matching R formatting exactly. **Names come from `DTFit`, no need to pass them**:

```haskell
let decisionTreeCls = df |-> decisionTree defaultDecisionTree { dtMaxDepth = Just 3 } ["Petal.Length","Petal.Width"] "species"
saveSVG "decisiontree-rpart.svg" $ toPlot decisionTreeCls
```

![Decision tree (rpart.plot style)](../images/decisiontree-rpart.svg)

To print split rules as **text**, use `printRpart decisionTreeCls` (R `print.rpart` format: `node) split n loss yval (yprob) *`). Low-level using matrix + names directly: `treePlotRaw` / `printRpartRaw` (`[Text] feature names -> [Text] class names -> DTree`).

## k-NN

Classification via `knnCls`, regression via `knnReg` (first arg = neighbor count k):

```haskell
knnCls :: Int -> [Text] -> Text -> KNNCSpec   -- classification → KNNClassifier (Plottable + ClassPredict)
knnReg :: Int -> [Text] -> Text -> KNNRSpec   -- regression → KNNRegressor
```

Classifier generic (applies hereafter): `confusionOf clf xTest yTest` (confusion matrix). `decisionBoundaryOf` (2-feature plane decision regions) grids an res×res region, predicts per cell, and **colors cells by class** (sklearn `DecisionBoundaryDisplay` equivalent; annotation-based). Training points overlay via `<> toPlot clf`, with fill colors matching the legend.

```haskell
import Hanalyze.Plot (toPlot, decisionBoundaryOf, confusionOf, knnCls, (|->))

let knnClsModel = df |-> knnCls 5 ["x1","x2"] "species"     -- KNNClassifier (k=5)
saveSVG "knn.svg"           $ decisionBoundaryOf knnClsModel (0, 8) (0, 4) 80 <> toPlot knnClsModel
saveSVG "knn-confusion.svg" $ confusionOf knnClsModel xTest yTest
```

![k-NN decision boundary](../images/knn-decision-boundary.svg)

![k-NN confusion matrix](../images/knn-confusion.svg)

**Low-level**: `fitKNNC :: Int -> Matrix Double -> VU.Vector Int -> KNNClassifier` /
`fitKNNR :: … -> VU.Vector Double -> KNNRegressor`. Prediction via `predictKNNC` / `predictKNNCProbs` /
`predictKNNR` (brute-force Euclidean distance). `confusionOf` works for any classifier with `ClassPredict`.

### Transparent standardization wrapper (`standardized` / `standardizedY`)

Distance-based models like k-NN suffer when features are not scaled uniformly; large-magnitude columns dominate the distance.
`standardized` transparently wraps a spec, fits in standardized space while **auto-inverse-transforming plots and predictions back to original scale** (tidymodels `step_normalize` / sklearn `Pipeline` equivalent; column names from the spec itself):

```haskell
import Hanalyze.Plot (toPlot, standardized, standardizedY, knnReg, lm, (|->))

let knnRegModel = df |-> standardized (knnReg 3 ["temp", "conc"] "rate")   -- standardize X; plots in original scale
    lmModel     = df |-> standardizedY (lm "temp" "rate")                  -- optionally standardize response too
```

`standardizedY` (X + response y) requires inverse-transforming predictions and CIs, so it's opt-in explicit. Meaningless or harmful specs compile but have `fitEither` return `Left`:

| spec | `standardized` | `standardizedY` |
|---|---|---|
| `knnReg` / `knnCls` (distance-based; **target use case**) | ✅ | regression only ✅ / classification returns `Left` |
| `lm` / `glm` / `spline` / `rlm` / `rq` / `lmMulti` / … (linear; normalization for interpretation) | ✅ | continuous-response regression only ✅ |
| `gp` / `regularized` / `pca` / `pls` (**internally pre-standardized**) | returns `Left` (avoid double standardization) | returns `Left` |
| `decisionTree` / `randomForestReg` / `gbmReg` (trees; **scale-invariant**) | returns `Left` (pointless) | returns `Left` |

- `glm` / `glmMulti` responses are scale-constrained by family/link (count / binary / positive), so `standardizedY` returns `Left` (only `standardized` for X available).
- Univariate (1 predictor) plots produce **original-scale scatter + prediction curve** automatically (`toPlot` handles inverse-transform internally, independent of inner standardized axes).

![Transparent standardization k-NN regression (original scale)](../images/knn-standardized.svg)

Linear models also support transparent standardization; regression line and CI bands emerge in original scale (bands also inverse-transformed):

![Transparent standardization LM (original scale + CI band)](../images/lm-standardized.svg)

Generic standardization utilities (composing `Standardizer` directly) are in [10 stat preprocessing section](10-stat.md#preprocessing--standardization--statstandardize).

## Naive Bayes

Classification only (continuous features = Gaussian NB). `NBModel` is `Plottable` (class mean scatter) + `ClassPredict`, so k-NN's `decisionBoundaryOf` / `confusionOf` work:

```haskell
naiveBayes :: [Text] -> Text -> NBSpec   -- classification → NBModel (Plottable + ClassPredict)
```

```haskell
let naiveBayesCls = df |-> naiveBayes ["x1","x2"] "species"   -- NBModel (= NBGaussian)
saveSVG "nb.svg" $ decisionBoundaryOf naiveBayesCls (0, 8) (0, 4) 60 <> toPlot naiveBayesCls
```

![Naive Bayes decision boundary (Gaussian)](../images/nb-decision-boundary.svg)

**Low-level**: `fitGNB :: Matrix Double -> VU.Vector Int -> GaussianNB` (continuous) /
`fitMNB :: Double -> Matrix Double -> VU.Vector Int -> MultinomialNB` (Laplace α, non-negative count matrix).
`NBModel = NBGaussian GaussianNB | NBMultinomial MultinomialNB`. Prediction via `predictNB` (labels) /
`predictNBLogProbs` (normalized log posteriors).

## SVM (Support Vector Machine)

Dual C-SVC only. Kernel choice via config (shared with GP / KRR from `Hanalyze.Model.Kernel`:
`Linear` / `Poly d` / `RBF` / `Matern52` / `Periodic`; default `Linear`). SMO is RNG-free, so fit is pure and deterministic.

Classification only (multi-class one-vs-rest; `SVMMulti`; SVR regression not provided yet; add when needed as `svmReg`). Holds `ClassPredict`, so `confusionOf` works:

```haskell
svmCls :: SVMConfig -> [Text] -> Text -> SVMSpec   -- classification → SVMMulti (ClassPredict)
```

Config `SVMConfig`:

```haskell
data SVMConfig = SVMConfig        -- defaultSVM
  { svmC         :: Double        -- regularization C (0 ≤ α ≤ C; larger = stricter violation penalty; default 1)
  , svmKernel    :: Kernel        -- kernel (default Linear)
  , svmParams    :: KernelParams  -- hyperparams (RBF/Poly γ = 1/(2ℓ²); Linear scale = σ_f²)
  , svmTol       :: Double        -- KKT tolerance               (default 1e-3)
  , svmMaxPasses :: Int           -- consecutive no-change pass limit (default 5)
  , svmMaxIter   :: Int           -- total pass limit (safety valve) (default 1000)
  , svmHyper     :: SVMHyper      -- hyperparameter selection method (default SVMFixed; see CV tuning)
  }
```

```haskell
import Hanalyze.Plot (svmCls, defaultSVM, SVMConfig (..), confusionOf, (|->))
import Hanalyze.Model.Kernel (Kernel (..), KernelParams (..), defaultKernelParams)

let rbfCfg = defaultSVM { svmKernel = RBF, svmParams = defaultKernelParams { kpLengthScale = 1.0 }, svmC = 10 }
    svmClsModel = df |-> svmCls rbfCfg ["x1","x2"] "species"   -- SVMMulti
```

Kernel SVM also renders **nonlinear decision regions** with `decisionBoundaryOf` (colors per class, like k-NN / Naive Bayes / MLP). Training points overlay.

![Kernel SVM (RBF) decision region and class-colored scatter](../images/svm-rbf-boundary.svg)

**CV auto-tuning**: Like GP/KRR, tuning is **baked into config** (no separate verb). Setting `svmHyper` to `SVMTuneCV grid` makes `svmCls` perform k-fold CV, selecting hyperparams for max accuracy and refitting (sklearn `GridSearchCV` / R `e1071::tune.svm` equivalent; fixed seed → deterministic):

```haskell
import Hanalyze.Plot (svmCls, defaultSVM, SVMHyper (..), SVMTuneGrid (..), defaultSVMTuneGrid, (|->))

let grid = defaultSVMTuneGrid { svmtCs = [0.1, 1, 10], svmtLengths = [0.5, 1, 2], svmtFolds = 5 }
    svmClsModel = df |-> svmCls defaultSVM { svmHyper = SVMTuneCV grid } ["x1","x2"] "species"  -- refits via CV
```

`SVMHyper = SVMFixed` (default; use config values) | `SVMTuneCV SVMTuneGrid` (CV search). `SVMTuneGrid` fields:
`svmtCs` (C candidates), `svmtKernels` (kernel candidates), `svmtLengths` (ℓ candidates), `svmtFolds` (fold count).
Matrix-direct: `tuneSVM cfg grid xMat y :: (SVMConfig, Double)` (best config, mean CV accuracy).

**Low-level** (`Hanalyze.Model.SVM`): 2-class `fitSVM cfg xMat yLab` (`yLab ∈ {0,1}`) /
multi-class `fitSVMMulti`. Prediction via `predictSVM` (labels) / `predictSVMScore` (decision score) / `predictSVMMulti`.
`numSupportVectors` counts SVs; `svmSupportVectorsOf` overlays SV markers (boundary line `decisionLineOf` and others in backlog).

## MDS (Multidimensional Scaling)

Dimension reduction preserving inter-sample distances to 2D. Methods: `MDSClassical` (Torgerson; equivalent to PCA for Euclidean distance) and `MDSSammon` (nonlinear, emphasizes small distances). Fits via `df |->` and returns model type `MDSResult`:

```haskell
import Hanalyze.Plot (mds, defaultMDS, MDSConfig (..), MDSMethod (..)
                            , toPlot, mdsView, mdsGroupBy, (|->))

let mdsModel = df |-> mds defaultMDS ["x1","x2","x3"]   -- MDSResult
saveSVG "mds.svg"   $ noDf |>> toPlot mdsModel                                   -- monochrome embedding
saveSVG "mds-g.svg" $ noDf |>> toPlot (mdsView mdsModel <> mdsGroupBy "species") -- species group colors

let mdsSammonModel = df |-> mds defaultMDS { mdsMethod = MDSSammon } ["x1","x2","x3"]  -- Sammon
```

Group color via `mdsGroupBy "columnName"` (`statColor` is `Color`-only, so separate column-name option). Low-level matrix/kernel versions (`mdsClassical` / `mdsSammon` / `euclideanDist`; `Hanalyze.Stat.MDS`) remain.

![MDS embedding scatter (classical, species group colors)](../images/mds-scatter.svg)

## Neural network (classic MLP)

Classification via `mlpCls`, regression via `mlpReg`. Internally uses seed-purified version, so fits via `df |->`. Classification NN is `ClassPredict`, so `decisionBoundaryOf` renders **nonlinear decision regions**. Loss curve `nnLossOf` confirms convergence:

```haskell
mlpCls :: MLPConfig -> Word32 -> [Text] -> Text -> MLPClsSpec   -- classification → MLPFit (ClassPredict)
mlpReg :: MLPConfig -> Word32 -> [Text] -> Text -> MLPRegSpec   -- regression → MLPFit
```

Config `MLPConfig` (shared between regression and classification):

```haskell
data MLPConfig = MLPConfig        -- defaultMLP
  { mlpHidden      :: [Int]       -- hidden layer unit counts      (default [16])
  , mlpActHidden   :: Activation  -- ReLU/Sigmoid/Tanh/Identity/Softmax (default ReLU)
  , mlpLR          :: Double      -- learning rate                 (default 0.01)
  , mlpEpochs      :: Int         -- number of epochs              (default 200)
  , mlpBatch       :: Int         -- mini-batch size               (default 16)
  , mlpL2          :: Double      -- L2 regularization             (default 0)
  , mlpStandardize :: Bool        -- z-score standardize X         (default True)
  }
```

```haskell
import Hanalyze.Plot (mlpCls, mlpReg, nnLossOf, decisionBoundaryOf, toPlot, (|->))

let mlpClsModel = df |-> mlpCls defaultMLP 42 ["x1","x2"] "species"   -- 42 = seed; MLPFit
saveSVG "nn-decision-boundary.svg" $ decisionBoundaryOf mlpClsModel (-4, 4) (-4, 4) 80
                                       <> layer (scatter "x1" "x2" <> colorBy "cls")
saveSVG "nn-loss.svg"              $ nnLossOf mlpClsModel
```

![MLP classification nonlinear decision region (separating concentric 2 classes)](../images/nn-decision-boundary.svg)

![NN training loss curve](../images/nn-loss.svg)

> ⚠ **Classic MLP only** (fully connected feedforward + backpropagation; mini-batch Adam + L2).
> CNN / RNN / Transformer, GPU, automatic differentiation frameworks out of scope. For tabular data nonlinear regression/classification.

**Low-level**: pure version `fitMLPClassifierPure cfg xMat yLab seed` / `fitMLPRegressorPure` (same seed → bit-identical). IO version `fitMLPClassifier … gen` (progress callback) also available.

MDS B matrix derivation, Sammon stress, NB theory are documented in [ml/usage-ml-extensions](../ml/usage-ml-extensions.md).
