# Principles of Hierarchical Bayesian Models (HBM)

> 🌐 **English** | [日本語](hbm.ja.md)

## Bayesian inference framework

**Posterior** = **likelihood** × **prior** / marginal likelihood

$p(\theta \mid y) \propto p(y \mid \theta) \cdot p(\theta)$

Bayesian linear regression:

$\alpha \sim \text{Normal}(0, 10),\ \beta \sim \text{Normal}(0, 10),\ \sigma \sim \text{Exponential}(1)$
$y_i \sim \text{Normal}(\alpha + \beta x_i, \sigma)$

## Why MCMC?

The posterior $p(\theta \mid y)$ usually has **no closed form**, so we approximate it
through **sample sequences** generated by Markov-chain Monte Carlo (MCMC).

## NUTS (No-U-Turn Sampler)

An evolution of Hamiltonian Monte Carlo (HMC):
- Generates pseudo-Hamiltonian trajectories with leapfrog.
- Auto-decides the trajectory length via the U-turn criterion → no tuning required.
- Default in Stan / PyMC.

hanalyze uses **AD (automatic differentiation)** to compute exact log-posterior gradients.

## Diagnostics

- **R-hat** < 1.01: convergence across chains.
- **ESS** (Effective Sample Size) > a few hundred: independent-information count after
  accounting for autocorrelation.
- **Trace plot**: visually inspect chain mixing.
- **Pair plot**: joint-posterior structure (correlations, funnels, …).

## Credible vs. confidence intervals

- **Confidence interval (frequentist)**: an interval interpretation under repeated experiments.
- **Credible interval (Bayesian)**: the 2.5–97.5 % percentile of the posterior →
  directly interpretable as "the true value lies in this interval with 95 % probability".
